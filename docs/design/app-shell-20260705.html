App shell wireframe

2026-07-05 · Phase 0 kickoff. Dashed = reserved for a later phase, not built in Phase 0/1.

Decisions

  • Single column everywhere. Multi-column deck rejected: conflicts with the one-handed / no-horizontal-scroll stories.
  • Timeline switcher (home / local / bubble / federated) is a tab row at the top of the column. Each tab keeps its own scroll position for the session.
  • Bottom tabs: Timeline / Notifications / Search (Phase 2, reserved) / own Profile. Favourites and bookmarks lists live as tabs on the own-profile screen (they are "my" API resources: /api/v1/favourites, /api/v1/bookmarks). Settings is a gear inside the profile screen.
  • Compose FAB: bottom-right thumb zone, reserved now so Phase 2 doesn't break the layout.

Open

  • Left swipe-in drawer: parked. It is an overlay, so it can be added later without reserving space. Watch for gesture conflict with a possible horizontal-swipe timeline switcher. Decide from dogfooding.
