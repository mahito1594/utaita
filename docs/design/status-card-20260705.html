Decisions

  • Zone order: boost line → header → reply-to line → CW bar → body + media (one block) → quote → reaction chips → action bar.
  • Boost renders as a header line on a single card. Nested inner card rejected: breaks the vertical rhythm of the timeline.
  • Reply-to line sits below the header but above the CW bar — it is metadata, not spoilable content, so it stays visible while the CW is collapsed.
  • Body and media form one contiguous block; the quote block goes below it. Quote above media rejected: a quote-reply with attached images would split the author's own content.
  • Reaction chips render image emoji (remote reactions have name@host + url) and highlight me.
  • Action bar (with counts) is visible from Phase 1, read-only until Phase 2. Included now because card height/spacing feeds the design tokens.

Field sources (verified against the reference instance, 2026-07-05)

  • reblog → boost line; in_reply_to_id / pleroma.in_reply_to_account_acct → reply-to line.
  • spoiler_text + sensitive → CW bar. content (HTML) + emojis → body. media_attachments → media grid.
  • quote / quote_id → quote block. emoji_reactions (count, name, url, me, account_ids) → chips.
  • replies_count / reblogs_count / favourites_count + visibility → action bar / header.
