# ユーザーストーリー (正本)

> この日本語版が正本。フェーズの区切りで英語版 [stories.md](./stories.md) に同期する。

形式: 1 ストーリー 1 行。自明でない場合のみ受け入れ条件を添える。フェーズごとにグループ化し、未割り当てのアイデアは Icebox へ。

> 状態: Phase 1 は 2026-07-07 のキックオフで精査済み。Phase 2 以降はロードマップから起こした初期ドラフトのままなので、pleroma-fe への実際の不満を観察して書き直していくこと。

## Phase 0 — 基盤

ユーザーストーリーの代わりに、一本の縦串を Done 条件とする:

- [x] dev proxy + 型生成済みクライアント経由でリファレンスインスタンスのタイムラインを取得し、アプリシェル (ラフ) 内に Status カードのリストとして描画できる。dev トークンは env から注入し、未設定なら 401/403 (エンドポイントで異なる — PLAN の落とし穴参照) がハンドリングされて UI に現れる。(2026-07-06 確認: 200 経路はブラウザ実機、401/403 はページテスト + 実インスタンスへの素の fetch で確認)

## Phase 1 — 閲覧 MVP

**Done 条件**: 下記ストーリーがすべてチェックされ、スマホ実機と PC の両方で成立している (片手での操作範囲、横スクロールなし)。かつ、閲覧については pleroma-fe を開かずに 1 日過ごせる。

- [ ] インスタンスのアカウントでログインでき、ブラウザを再起動してもログイン状態が保持される。ログアウトすると確実に未認証状態に戻る (Akkoma は Bearer リクエストにセッション Cookie も返す — PLAN の落とし穴参照)。
- [ ] Home タイムラインを滑らかな無限スクロールで読める。CW を開いてもスクロール位置がずれない。
- [ ] 開きっぱなしのタイムラインで、ページを再読み込みせずに新着を読み込める。
  - 手動更新 (ボタンないしプルリフレッシュ)。前方取得 (`min_id`) のロジックは UI から分離しておく — Phase 4 の Streaming/自動更新トグルが同じ合流処理を使う
- [ ] home / local / bubble / federated を切り替えて読める。
- [ ] Status カードで投稿の中身が一通り読める: 本文 (HTML)、添付メディア、CW/sensitive、カスタム絵文字、ブースト元、絵文字リアクション、投票、リンクプレビュー。
  - 本文はサニタイズ済み HTML として描画。`<small>` など pleroma-fe が取りこぼす基本タグも正しく表示する。MFM 専用描画には対応しない (`content` の HTML をそのまま描画)
  - メディアは 2 列グリッドで全枚数表示 (1 枚なら全幅)、タップで簡易オーバーレイ原寸。動画/音声はネイティブ `<video>`/`<audio>`。スワイプ送り付きライトボックスは Icebox。リモート添付の `blurhash`/`meta` 欠落で落ちない。sensitive メディアはデフォルトで隠れる
  - 投票は閲覧のみ (選択肢・票数・締切/終了状態)。リンクプレビューはタイトル+サムネイルの軽量版
  - 本文中のメンションはアプリ内のプロフィールへ。外部リンクは新しいタブで開く (ADR-0011)
  - 引用は深さ 1 のミニカードとして本文下に描画 (2026-07-13 決定 — ADR-0007)。描画時は本文の `span.quote-inline` (サーバ付加の RE: リンク) を除去。`quote` が null (未取得リモート等) やミニカード内のさらなる引用は RE: リンクのまま残す
  - 絵文字リアクション・ブースト・ファボの「誰が」一覧はカードには置かず、投稿詳細 (スレッド) ビューの持ち物 (会話ツリーのストーリーで実装)
- [ ] 投稿を開いて会話ツリー全体を読める。どの投稿から来たのか見失わない。
  - 親がインスタンス未取得 (`in_reply_to_id: "_"` — PLAN の落とし穴参照) の場合は resolve (`/api/v2/search?resolve=true`) を試みてアプリ内でツリーに繋げる。それでも取得できない場合はプレースホルダーを表示し、元インスタンスへのリンクは外部と明示した副次的な導線としてのみ添える (ADR-0011)
  - 絵文字リアクション・ブースト・ファボの「誰が」一覧をここで開ける (`/api/v1/pleroma/statuses/:id/reactions`、`/reblogged_by`、`/favourited_by` — 形はセッション冒頭に実測)
- [ ] プロフィールを投稿 / 返信 / メディアのタブ付きで閲覧できる。フォロー関係 (フォロー中 / されている) が見える。
- [ ] 通知を読める。メンション・ブースト・フォロー・絵文字リアクションが一目で区別でき、`pleroma:emoji_reaction` や `move` など未知の type で落ちない。

## Phase 2 — 書き込み

- [ ] CW・公開範囲 (`local` 含む) 付きで投稿でき、カスタム絵文字の補完が効く。
- [ ] alt text 付きで画像を添付でき、アップロードの進捗が見える。
- [ ] タイムライン上からファボ・ブースト・ブックマーク・絵文字リアクションができる。
- [ ] プロフィールからフォロー / 解除でき、承認制アカウントには申請中の状態が表示される。
- [ ] アカウント・ハッシュタグ・投稿を検索できる。
- [ ] お気に入り / ブックマークした投稿の一覧を閲覧できる (`/api/v1/favourites`, `/api/v1/bookmarks`)。

## Icebox

- 見逃した投稿のキャッチアップダイジェスト (Phanpy 風)
- タブを閉じても消えない下書き
- Streaming によるタイムラインのリアルタイム更新 — 自動更新 / 手動更新をトグルで切り替えられる形で (昔の Twitter クライアント風)
- 複数アカウント
- タイムラインごとのスクロール位置保持 (Phase 1 キックオフで削除 — home 常駐の利用実態では益が薄い。dogfooding で必要になったら)
- スワイプ送り・ピンチズーム付きのメディアライトボックス
- ハッシュタグタイムライン (`/api/v1/timelines/tag/{tag}`)
- リストタイムライン (`/api/v1/timelines/list/{list_id}`)
- DM の閲覧 (spec のタイムライン一覧に direct が無い — Akkoma の chats 機構の調査から)
- リモートにしか存在しない返信の補完 (発言元インスタンスの公開 API をマージする差別化機能。同一オリジン前提と衝突するため要設計)
